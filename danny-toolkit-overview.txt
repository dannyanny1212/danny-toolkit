  Performance Optimalisatie: Heartbeat Daemon, CorticalStack, SwarmEngine

PROBLEEM:
  De Heartbeat Daemon (app 52) was traag door:
  - Blokkerende asyncio.run() in de main UI loop (5-30s freezes)
  - Synchrone SQLite commits na elke individuele write (lock contention)
  - 34+ threads door onbegrensde thread pool (GIL contention)
  - Te frequente swarm checks (elke 10s) en stat logging (elke 5s)
  - Security scan blokkeerde de UI 30-60 seconden per uur

GEWIJZIGDE BESTANDEN (code):
────────────────────────────────────────────────────────────────────────────────

danny_toolkit/daemon/heartbeat.py
  WAT VERANDERD:
  - Swarm taken draaien nu in een 2-worker ThreadPoolExecutor (was: asyncio.run()
    blokkeerde de main loop). Nieuwe methode _execute_swarm_task() draait in
    worker thread met eigen asyncio event loop.
  - Security scan (volledig_rapport()) draait nu in background thread via
    _execute_security_scan(). Was: inline in main loop, 30-60s freeze.
  - Swarm schedule check interval: 10 → 60 pulsen (83% minder checks)
  - Stat logging interval: 5 → 30 pulsen (83% minder DB writes)
  - Worker pool wordt netjes afgesloten bij stop() en KeyboardInterrupt
  - CorticalStack wordt geflusht (stack.flush()) bij daemon shutdown
  - Nieuwe imports: queue, ThreadPoolExecutor
  - Nieuwe attributen: _task_queue (Queue), _worker_pool (ThreadPoolExecutor)

danny_toolkit/brain/cortical_stack.py
  WAT VERANDERD:
  - Writes worden nu gebatched: flush na 20 writes OF na 5 seconden
    (was: commit() na elke individuele write)
  - PRAGMA synchronous=NORMAL toegevoegd (minder fsync overhead, veilig met WAL)
  - Nieuwe class constanten: _BATCH_SIZE = 20, _BATCH_INTERVAL = 5.0
  - Nieuwe attributen: _pending_writes (int), _last_flush (float)
  - Nieuwe private methode: _maybe_flush() — conditionale commit binnen lock
  - Nieuwe publieke methode: flush() — forceer commit van alle pending writes
  - close() flusht nu pending writes voordat de connectie sluit
  - Alle log_event(), remember_fact(), recall(), log_stat() gebruiken nu
    _maybe_flush() in plaats van directe self._conn.commit()

swarm_engine.py
  WAT VERANDERD:
  - Nieuwe constante: _SWARM_MAX_WORKERS = 6 (was: onbegrensd, ~34 threads)
  - In run(): per event loop wordt een verse ThreadPoolExecutor(max_workers=6)
    als default executor ingesteld. Voorkomt GIL contention en context switching.
  - Nieuwe import: ThreadPoolExecutor uit concurrent.futures
  - Opmerking: executor wordt per-loop aangemaakt (niet module-level gedeeld)
    omdat asyncio.run() de default executor shutdown bij afsluiting.
