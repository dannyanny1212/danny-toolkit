LLM PROMPTING - PROMPT ENGINEERING - AI INSTRUCTIES

=== WAT IS PROMPTING? ===

Prompting is de kunst van het formuleren van instructies voor Large Language Models (LLMs) zoals Claude, GPT en anderen. Een goede prompt maakt het verschil tussen een nutteloos en een briljant antwoord.

Keywords: prompting, prompt engineering, LLM, instructies, AI

---

=== BASIS PROMPTING PRINCIPES ===

1. WEES SPECIFIEK
Slecht: "Schrijf iets over Python"
Goed: "Schrijf een Python functie die een lijst sorteert met bubble sort, inclusief comments"

2. GEEF CONTEXT
- Wie ben je? (rol)
- Wat is het doel?
- Voor wie is de output?
- Welke beperkingen zijn er?

3. GEEF VOORBEELDEN
- Few-shot learning: geef 2-3 voorbeelden
- Dit helpt het model het patroon te begrijpen

Keywords: specifiek, context, voorbeelden, duidelijke instructies

---

=== WAT IS ZERO-SHOT PROMPTING? ===

Zero-shot prompting is een vraag stellen zonder voorbeelden te geven. Het model moet de taak begrijpen uit alleen de instructie.

Voorbeeld:
"Vertaal naar het Engels: Hallo wereld"

Wanneer gebruiken:
- Simpele taken
- Model kent het format al
- Snelle interactie

Keywords: zero-shot, geen voorbeelden, directe vraag

---

=== WAT IS FEW-SHOT PROMPTING? ===

Few-shot prompting geeft het model voorbeelden voor de vraag. Dit helpt het model het gewenste format te begrijpen.

Voorbeeld:
"positief: Ik ben blij -> sentiment: positief
negatief: Dit is verschrikkelijk -> sentiment: negatief
Wat een mooie dag! -> sentiment:"

Wanneer gebruiken:
- Specifiek output format nodig
- Ongebruikelijke taken
- Consistente resultaten gewenst

Keywords: few-shot, voorbeelden, examples, in-context learning

---

=== WAT IS CHAIN-OF-THOUGHT PROMPTING? ===

Chain-of-Thought (CoT) vraagt het model stap voor stap te denken. Dit verbetert redenering bij complexe problemen.

Voorbeeld:
"Los dit wiskundeprobleem stap voor stap op:
Een trein rijdt 120 km in 2 uur. Wat is de snelheid?"

Stappen in antwoord:
1. Snelheid = afstand / tijd
2. Snelheid = 120 km / 2 uur
3. Snelheid = 60 km/uur

Wanneer gebruiken:
- Wiskundige problemen
- Logische redenering
- Multi-step taken

Keywords: chain-of-thought, CoT, stap voor stap, redeneren, denken

---

=== WAT IS ROLE PROMPTING? ===

Role prompting geeft het model een persona of rol. Dit beinvloedt de stijl en expertise van antwoorden.

Voorbeelden:
- "Je bent een ervaren Python developer..."
- "Gedraag je als een vriendelijke leraar..."
- "Je bent een security expert die code reviewt..."

Waarom role prompting werkt:
- Activeert relevante kennis
- Zet juiste toon
- Beperkt scope van antwoord

Keywords: role prompting, persona, rol, expert, karakter

=== PROMPT TEMPLATES ===

INSTRUCTIE TEMPLATE:
"""
Je bent [ROL].
Je taak is om [TAAK].
Houd rekening met [BEPERKINGEN].
Formatteer je antwoord als [FORMAT].

[INPUT]
"""

ANALYSE TEMPLATE:
"""
Analyseer de volgende [TYPE]:

[CONTENT]

Geef:
1. Samenvatting (max 2 zinnen)
2. Hoofdpunten (bullets)
3. Conclusie
"""

CODE TEMPLATE:
"""
Schrijf een [TAAL] functie die:
- Input: [BESCHRIJVING]
- Output: [BESCHRIJVING]
- Randgevallen: [LIJST]

Voeg comments toe.
"""

=== VEELGEMAAKTE PROMPTING FOUTEN ===

1. Te vaag zijn
- Model moet niet gokken wat je wilt

2. Te veel in een prompt
- Splits complexe taken op

3. Tegenstrijdige instructies
- "Wees kort maar geef veel detail"

4. Geen format specificeren
- Je krijgt inconsistente outputs

---

=== PROMPTING TIPS ===

- Itereer: verfijn prompt gebaseerd op output
- Test met variaties van je vraag
- Gebruik delimiters (""", ---, ###)
- Vraag om verificatie: "Controleer je antwoord"
- Geef negatieve voorbeelden: "Doe NIET..."
- Temperature: laag voor consistent, hoog voor creatief

Keywords: tips, best practices, verbeteren, itereren
