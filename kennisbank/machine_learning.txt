MACHINE LEARNING - KUNSTMATIGE INTELLIGENTIE - ML - AI

=== WAT IS MACHINE LEARNING? ===

Machine Learning (ML) is een tak van kunstmatige intelligentie (AI) waarbij computers leren van data zonder expliciet geprogrammeerd te worden. In plaats van regels te schrijven, geef je het systeem voorbeelden en het ontdekt zelf patronen.

Keywords: machine learning, ML, kunstmatige intelligentie, AI, leren van data, patronen herkennen

---

=== WAT IS SUPERVISED LEARNING? ===

Supervised Learning (begeleid leren) is een type machine learning waarbij je het model gelabelde data geeft. Dit betekent: input data samen met het correcte antwoord (label).

Hoe werkt supervised learning:
- Je geeft het model voorbeelden met bekende antwoorden
- Het model leert de relatie tussen input en output
- Daarna kan het voorspellingen maken voor nieuwe data

Voorbeelden van supervised learning:
- Spam detectie (email is spam of niet)
- Huizenprijzen voorspellen
- Beeldherkenning (foto van kat of hond)
- Medische diagnose

Supervised learning algoritmes:
- Linear Regression (voor continue waarden)
- Logistic Regression (voor classificatie)
- Decision Trees
- Random Forest
- Neural Networks
- Support Vector Machines (SVM)

Keywords: supervised learning, begeleid leren, gelabelde data, classificatie, regressie, voorspellen

---

=== WAT IS UNSUPERVISED LEARNING? ===

Unsupervised Learning (onbegeleid leren) is machine learning zonder labels. Het model krijgt alleen data zonder correcte antwoorden en moet zelf structuur ontdekken.

Hoe werkt unsupervised learning:
- Model ontvangt alleen input data
- Zoekt zelf naar patronen en groepen
- Geen "correcte" antwoorden om van te leren

Voorbeelden van unsupervised learning:
- Klantensegmentatie (groepen klanten vinden)
- Anomalie detectie (uitschieters vinden)
- Topic modeling (onderwerpen in tekst)
- Dimensionaliteitsreductie

Unsupervised learning algoritmes:
- K-Means Clustering
- Hierarchical Clustering
- PCA (Principal Component Analysis)
- Autoencoders
- DBSCAN

Keywords: unsupervised learning, onbegeleid leren, clustering, patronen, segmentatie

---

=== WAT IS REINFORCEMENT LEARNING? ===

Reinforcement Learning (bekrachtigingsleren) is machine learning waarbij een agent leert door interactie met een omgeving. De agent krijgt beloningen of straffen voor acties.

Hoe werkt reinforcement learning:
- Agent neemt actie in omgeving
- Omgeving geeft feedback (beloning/straf)
- Agent leert optimale strategie (policy)
- Doel: maximaliseer totale beloning

Voorbeelden van reinforcement learning:
- Game AI (schaken, Go, videogames)
- Robotica (lopen, grijpen)
- Zelfrijdende auto's
- Aanbevelingssystemen

Reinforcement learning algoritmes:
- Q-Learning
- Deep Q-Networks (DQN)
- Policy Gradient
- PPO (Proximal Policy Optimization)
- Actor-Critic

Keywords: reinforcement learning, bekrachtigingsleren, agent, beloning, policy, omgeving

---

=== TRAINING VS INFERENCE ===

Training: Het proces waarbij het model leert van data
- Kan uren of dagen duren
- Vereist veel rekenkracht (GPU)
- Gebeurt eenmalig of periodiek

Inference: Het getrainde model gebruiken voor voorspellingen
- Duurt milliseconden
- Kan op gewone hardware
- Gebeurt continu in productie

---

=== WAT IS OVERFITTING? ===

Overfitting is wanneer een machine learning model de trainingsdata "uit het hoofd" leert in plaats van algemene patronen te ontdekken. Het model presteert uitstekend op trainingsdata maar faalt op nieuwe, ongeziene data.

Hoe herken je overfitting:
- Training accuracy is hoog (bijvoorbeeld 99%)
- Test/validatie accuracy is veel lager (bijvoorbeeld 70%)
- Model is te complex voor de hoeveelheid data
- Model vangt ruis op in plaats van signaal

Oorzaken van overfitting:
- Te weinig trainingsdata
- Model heeft te veel parameters
- Te lang getraind (te veel epochs)
- Geen regularisatie toegepast

Oplossingen voor overfitting:
- Meer trainingsdata verzamelen
- Regularisatie (L1, L2)
- Dropout in neural networks
- Early stopping
- Cross-validation gebruiken
- Simpeler model kiezen

Keywords: overfitting, overfitten, te complex, generalisatie, trainingsdata

---

=== WAT IS UNDERFITTING? ===

Underfitting is wanneer een machine learning model te simpel is om de patronen in de data te leren. Het presteert slecht op zowel training- als testdata.

Hoe herken je underfitting:
- Training accuracy is laag
- Test accuracy is ook laag
- Model mist duidelijke patronen

Oplossingen voor underfitting:
- Complexer model gebruiken
- Meer features toevoegen
- Langer trainen
- Minder regularisatie

Keywords: underfitting, te simpel, patronen missen

=== EVALUATIE METRICS ===

Classificatie metrics:
- Accuracy: percentage correcte voorspellingen
- Precision: van positieve voorspellingen, hoeveel correct?
- Recall: van echte positieven, hoeveel gevonden?
- F1-Score: harmonisch gemiddelde van precision en recall

Regressie metrics:
- MSE (Mean Squared Error)
- MAE (Mean Absolute Error)
- R-squared (verklaarde variantie)
